# Decision 4 — CLI grammar, dispatch & exit-code taxonomy

**Decision.** idgen is a single command with flags (no subcommands); `--decode`
flips mode. One `flag.FlagSet` is parsed in `Run`, then branched. `Run` takes the
build-stamped version string as a parameter (`main` owns the linker seam, D6, and
threads it in) — the CLI never reads a version literal of its own:

```go
// Run(args, stdin, stdout, stderr, version, clock) int
fs := flag.NewFlagSet("idgen", flag.ContinueOnError)
fs.SetOutput(stderr)
var (
    decode  bool   // --decode
    number  int    // -n / --number, default 1
    prefix  string // -p / --prefix, default "R"
    showVer bool   // --version / -V
)
// -n/--number, -p/--prefix, and --version/-V each bound twice (long + short).
// Parse itself calls fs.Usage before returning ErrHelp (and after printing a
// bad-flag error), so Usage must be a no-op — Run owns every usage print, or
// --help would emit the text twice (once from Parse, once from the ErrHelp case).
fs.Usage = func() {}

switch err := fs.Parse(args); {
case errors.Is(err, flag.ErrHelp): writeUsage(stdout); return exitSuccess // --help/-h
case err != nil:                   writeUsage(stderr); return exitUsage   // bad flags → 2 (flag already wrote its error line to stderr)
}
if showVer { fmt.Fprintln(stdout, version); return exitSuccess } // version is the injected param
if decode  { return runDecode(fs.Args(), stdin, stdout, stderr) }
return runMint(number, prefix, fs.NArg(), stdout, stderr, clk)
```

- **Stdlib `flag`**, no third-party CLI lib (research: no external deps). `flag`
  requires flags before positionals — matches the product's `idgen --decode
  R-...` examples.
- **`--help`/`-h`** → usage to **stdout**, exit 0. **`--version`/`-V`** → the bare
  injected version string to stdout, exit 0 (where the string comes from: D6).
- **Mint takes no positionals**: `runMint` errors if `NArg() != 0` — writes
  `idgen: unexpected argument(s): <args>` to stderr and returns exit 2 (no
  silent exit — see README Conventions).
- **`-n`/`-p` in decode mode: accepted but ignored** (mint concerns, no decode
  meaning; keeps the grammar simple).
- **Exit codes:** `0` success · `2` usage error (bad flags, empty/invalid
  prefix, `number ≤ 0`, mint-with-positionals) · `1` decode data failure (≥1
  malformed id in an otherwise valid invocation). Constants `exitSuccess=0`,
  `exitFailure=1`, `exitUsage=2`.

**Rejected.**
- *Subcommands (`idgen mint`/`idgen decode`).* Contradicts the documented
  default-action-plus-`--decode` UX.
- *Single non-zero code for everything.* The 1-vs-2 split
  separates "called it wrong" from "a batch value was bad" at no cost.
- *Error on `-n`/`-p` in decode mode.* Gold-plating the product doesn't ask for.

**Verification.** Table-driven `Run` tests over arg vectors assert (stdout, stderr,
exit code):
- `R-X0NT-UVTW` — `--help`/`-h` → usage **exactly once** on **stdout** **and
  stderr is empty**, exit 0 — rules out the stdlib-`flag` double print, where
  `Parse` auto-calls `fs.Usage` before returning `ErrHelp` and the text lands on
  both streams. The test must **count** occurrences of the `Usage:` line in stdout
  and assert the count is **exactly 1** (asserting only "stdout is non-empty"
  passes a double print and does not satisfy this id).
- `R-X1VQ-8NKL` — `--version` → the injected version string on **stdout**, exit 0
  (the test passes a known version into `Run` and asserts stdout is exactly it).
- `R-TIQT-QVU3` — `-V` (the short alias) → the same injected version string on
  **stdout**, exit 0 — the alias is wired, not a silently-ignored flag.
- `R-X33M-MFBA` — unknown flag → exit 2, stderr non-empty (flag's own parse
  error).
- `R-X4BJ-071Z` — mint with a positional argument → exit 2 **and** stderr is
  non-empty (names the unexpected argument) — not merely a non-zero exit.
- `R-X5JF-DYSO` — `--decode` routes to the decode path.
- `R-X6RB-RQJD` — `-n`/`-p` supplied with `--decode` are inert (decode output
  unchanged).
- `R-7UL7-PF0O` — Bare invocation (`idgen`, no flags, no positionals): stdout is
  **exactly one line** matching `^R-[0-9A-Z]{4}-[0-9A-Z]{4}$` — the default
  action mints once with the default `R` prefix, never erroring and never
  requiring `-p`.
- `R-PU67-68HE` — Mint with `-p X` (both a 1-char and a multi-char custom
  prefix): stdout is **exactly one line** matching `^X-[0-9A-Z]{4}-[0-9A-Z]{4}$`
  — the supplied prefix *replaces* the default outright, so the default `R`
  never appears anywhere in the output (rules out a concatenation bug that
  would print `XR-...`).
