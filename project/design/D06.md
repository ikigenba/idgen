# Decision 6 — Version, usage text, Makefile & release

**Decision.** The version follows the sibling `embed` model exactly: **the git
tag is the single source of truth, stamped into the binary at build time via a
linker seam**. There is no version literal anywhere in the sources.

**Version linker seam.** `package main` declares the seam; nothing else holds a
version:

```go
// cmd/idgen/main.go
var version = "dev" // overridden at build time via -ldflags "-X main.version=..."
```

`main` threads this value into `cli.Run(args, stdin, stdout, stderr, version,
clock)` (D4); `--version`/`-V` print whatever `version` holds, verbatim, never
parsing or reformatting it. A build on a clean semver tag reports the tag
(`v0.1.0`); a build past a tag reports the describe form (`v0.1.0-3-gabc1234`);
modified sources append `-dirty`; a build with no ldflags — **including every
`go test` binary** — reports `dev`. `internal/cli/version.go` (the old
hand-maintained `var version = "v0.1.0"`) is **deleted**.

**Usage text** lives in `internal/cli/usage.go` (`func writeUsage(io.Writer)`),
to stdout for `--help`/`-h`, to stderr on a usage error; covers default mint,
`-n/--number`, `-p/--prefix`, `--decode` (args-or-stdin note), `--help`,
`--version`/`-V`.

**Makefile** — mirrors `embed`'s, adapted to `idgen`. It stamps the git-derived
version on `build` and installs the built binary under `PREFIX`:

```make
BINARY  := idgen
BIN_DIR := bin
PREFIX  ?= $(HOME)/.local

.PHONY: build fmt test install clean

build: $(BIN_DIR)/$(BINARY)

$(BIN_DIR)/$(BINARY): go.mod $(shell find cmd internal -name '*.go' 2>/dev/null)
	install -d $(BIN_DIR)
	go build -ldflags "-X main.version=$(shell git describe --tags --always --dirty)" -o $(BIN_DIR)/$(BINARY) ./cmd/idgen

fmt:
	go fmt ./...

test:
	go test ./...

install: build
	install -d $(PREFIX)/bin
	install -m 0755 $(BIN_DIR)/$(BINARY) $(PREFIX)/bin/$(BINARY)

clean:
	rm -rf $(BIN_DIR)
	go clean
```

(`go.sum` is not a prerequisite: idgen has no third-party dependencies, so none
is generated — the sibling lists it only because it has one. The green gate is
still `go test -race ./...`, per Conventions; the Makefile's `test` target is the
plain `go test ./...`, matching the sibling.)

**Release — pushing a `vMAJOR.MINOR.PATCH` tag.** Three repo-root files, mirrored
from `embed`, publish a release, targeted at the actual GitHub remote
`ikigenba/idgen` (independent of the Go module path):

- **`.goreleaser.yaml`** (goreleaser v2):

  ```yaml
  version: 2
  project_name: idgen
  builds:
    - id: idgen
      main: ./cmd/idgen
      binary: idgen
      env:
        - CGO_ENABLED=0
      goos: [linux, darwin]
      goarch: [amd64, arm64]
      ldflags:
        - -s -w -X main.version={{ .Tag }}
  archives:
    - id: idgen
      formats: [tar.gz]
      name_template: "{{ .Binary }}_{{ .Os }}_{{ .Arch }}"
  checksum:
    name_template: "checksums.txt"
  release:
    github:
      owner: ikigenba
      name: idgen
  changelog:
    use: github
    sort: asc
  ```

  The `-X main.version={{ .Tag }}` stamp matches the local `git describe` scheme
  (both carry the leading `v`). Archive names are versionless
  (`idgen_<os>_<arch>.tar.gz`) so `install.sh` can fetch a stable
  `releases/latest/download/...` URL with no API call.

- **`.github/workflows/release.yml`** — mirrors `embed`'s verbatim (generic; no
  project-specific values): `on: push: tags: ['v*']`, `permissions: contents:
  write`, one `release` job on `ubuntu-latest` that checks out with
  `fetch-depth: 0`, sets up Go `1.26`, and runs `goreleaser-action@v6` with
  `args: release --clean` and `GITHUB_TOKEN`.

- **`install.sh`** — mirrors `embed`'s installer with the identifiers swapped:
  `REPO="ikigenba/idgen"`, `BINARY="idgen"`, version-override env var
  `IDGEN_VERSION` (default `latest`), destination
  `${BINDIR:-${PREFIX:-$HOME/.local}/bin}`. It maps `uname` to the
  `linux|darwin` / `amd64|arm64` matrix, downloads
  `releases/{latest/download | download/$VERSION}/idgen_<os>_<arch>.tar.gz`,
  untars, and `install -m 0755`s the binary onto `PATH`.

**Rejected.**

- *A hand-maintained version literal in source (`var version = "v0.1.0"`, or a
  `VERSION` file).* It is a second source of truth that drifts from the tag; the
  linker seam makes git the only owner. This reverses the prior decision — the
  project has deliberately adopted the sibling `embed`'s provenance model.
- *A separate `internal/version` package.* Over-engineered for one string; the
  seam is a single `var` in `main`, where `-ldflags -X` must target it.
- *`runtime/debug.ReadBuildInfo` fallback.* A second, less predictable version
  path for a binary always built via the Makefile or goreleaser; `dev` is an
  honest, explicit answer for an unstamped build.
- *Source-only distribution (the prior scope).* The product now promises a
  prebuilt binary; goreleaser + `install.sh` are the mechanism, matching the
  sibling so the two tools install the same way.

**Verification.**
- `R-TJYQ-4NKS` — an unstamped build (no ldflags — the `go test` binary itself,
  whose `main.version` defaults to `dev`) running `--version`/`-V` writes exactly
  `dev` (plus newline) to stdout and returns exit 0.
- `R-TL6M-IFBH` — building the real `cmd/idgen` package with
  `go build -ldflags "-X main.version=<sentinel>"` and executing the produced
  binary with `--version` writes exactly the sentinel (plus newline) to stdout —
  this exercises the **real linker seam** end to end (a genuine build + run), not
  the in-process default.
- `R-XLE4-CZFP` — `--help` asserts usage mentions each flag (`-n`, `-p`,
  `--decode`).
- `R-XMM0-QR6E` — build smoke: `go test ./...` is green and `go build` produces
  `bin/idgen` (the Makefile's `build`/`test` targets exercised).
